# Contributing to Super Agent Dashboard

Thank you for your interest in contributing! This guide will help you get started.

---

## Getting Started

### 1. Fork & Clone

```bash\ngit clone https://github.com/yourusername/super-agent-web.git\ncd super-agent-web\n```\n\n### 2. Install Dependencies\n\n```bash\npnpm install\n```\n\n### 3. Create Feature Branch\n\n```bash\ngit checkout -b feature/your-feature-name\n```\n\n---\n\n## Development Workflow\n\n### 1. Make Changes\n\n- Write code following our style guide\n- Add tests for new functionality\n- Update documentation\n\n### 2. Run Tests\n\n```bash\npnpm test              # Unit tests\npnpm test:integration  # Integration tests\npnpm lint             # Linting\npnpm check            # TypeScript check\n```\n\n### 3. Commit Changes\n\n```bash\ngit add .\ngit commit -m \"feat: add new feature\"\n```\n\nUse [Conventional Commits](https://www.conventionalcommits.org/):\n- `feat:` New feature\n- `fix:` Bug fix\n- `docs:` Documentation\n- `style:` Code style\n- `refactor:` Refactoring\n- `test:` Tests\n- `chore:` Maintenance\n\n### 4. Push & Create PR\n\n```bash\ngit push origin feature/your-feature-name\n```\n\nCreate a Pull Request with:\n- Clear description of changes\n- Reference to related issues\n- Screenshots (if UI changes)\n- Test coverage\n\n---\n\n## Code Style\n\n### TypeScript\n\n- Use strict mode\n- Avoid `any` types\n- Use interfaces for objects\n- Document complex logic\n\n### React\n\n- Functional components only\n- Use hooks for state\n- Prop types with TypeScript\n- Memoize expensive computations\n\n### Database\n\n- Use Drizzle ORM\n- Parameterized queries\n- Proper indexing\n- Migration files\n\n---\n\n## Testing Requirements\n\n- Minimum 80% code coverage\n- Unit tests for all functions\n- Integration tests for workflows\n- E2E tests for critical paths\n\n---\n\n## Documentation\n\n- Update README for major changes\n- Add JSDoc comments\n- Update API documentation\n- Add examples for new features\n\n---\n\n## Security\n\n- No hardcoded secrets\n- Validate all inputs\n- Use parameterized queries\n- Follow OWASP guidelines\n\n---\n\n## Performance\n\n- Optimize database queries\n- Minimize bundle size\n- Use lazy loading\n- Cache appropriately\n\n---\n\n## Review Process\n\n1. **Automated Checks**: CI/CD pipeline runs\n2. **Code Review**: Team reviews code\n3. **Approval**: At least 2 approvals required\n4. **Merge**: Squash and merge to main\n\n---\n\n## Questions?\n\nOpen an issue or contact us at: support@example.com\n\n---\n\n**Thank you for contributing!**\n
